Main Project

Workiwi

팀 전용 생성형 AI 협업 워크스페이스

문제 정의

소규모 팀 프로젝트 진행 시, 팀원들이 각자 다른 AI 툴을 사용하여 프롬프트와 컨텍스트가 파편화되는 문제가 발생했습니다. 이로 인해 결과물의 톤앤매너가 불일치하고, 중요한 기획 의도가 팀 전체에 공유되지 못해 협업 효율이 저하되었습니다.

해결책

팀 단위의 단일화된 AI 컨텍스트(Single Source)를 제공하는 워크스페이스를 구축했습니다. 프로젝트의 룰셋을 백엔드에서 중앙 관리하여 AI에게 주입하고, 채팅 내용을 즉시 구조화된 문서로 변환하여 자산화했습니다.

시연 영상

Workiwi 실제 구동 및 시연 영상

Workiwi 메인 대시보드 화면

시스템 아키텍처 & 설계

React 기반의 SPA로 프론트엔드를 구성하고, Node.js/Express 백엔드를 통해 Gemini API와 통신하는 구조입니다.

  • Context Injection Layer: 사용자 질문 도달 전, 백엔드 미들웨어에서 프로젝트별 '룰셋'을 시스템 프롬프트로 결합.
  • In-Memory Storage: 빠른 프로토타이핑 검증을 위해 메모리 기반 저장소 구축 (향후 DB 확장 고려한 JSON 설계).
Key Feature 01

프로젝트 컨텍스트 주입 &
역할별 에이전트

사용자가 별도의 설명 없이 질문하더라도 AI가 프로젝트의 기술 스택, 코딩 컨벤션, 톤앤매너를 이미 알고 답변하도록 구현했습니다.

1

동적 프롬프트 주입

백엔드에서 룰셋을 조회하여 System Prompt로 변환

2

페르소나 모드

PM, Dev, Designer 등 직무별 최적화된 답변 제공

Key Feature 02

채팅의 자산화
(Chat-to-Doc 파이프라인)

휘발되기 쉬운 AI와의 대화 내용을 클릭 한 번으로 구조화된 JSON 문서로 변환하여 저장합니다.
